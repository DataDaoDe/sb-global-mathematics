# Web Data Format

The static website data bundle is generated by:

```bash
pnpm build:web-data
```

The source YAML remains authoritative. Files under `generated/web/` are replaceable artifacts for browser and static-site consumers.

## Files

```text
generated/web/
├── metadata.json
├── index.json
├── search.json
├── tree.json
└── entities/
    └── <entity-id>.json
```

## Metadata

`metadata.json` contains:

```json
{
  "schema_version": 1,
  "generated_at": "2026-06-20T00:00:00.000Z",
  "entity_count": 10,
  "edge_count": 24
}
```

Consumers must check `schema_version` before assuming compatibility.

## Index

`index.json` is a sorted list of entity summaries:

```json
{
  "id": "algebra.ring.commutative-unital",
  "kind": "concept",
  "title": "Commutative unital ring"
}
```

## Search

`search.json` is a lightweight text index:

```json
{
  "id": "algebra.ring.commutative-unital",
  "kind": "concept",
  "title": "Commutative unital ring",
  "text": "Commutative unital ring\nAn associative ring..."
}
```

This is not a full search engine. It is a portable starter index for static clients.

## Namespace Tree

`tree.json` is a namespace hierarchy derived from entity identifiers:

```json
{
  "root": {
    "path": "",
    "label": "Mathematics",
    "depth": 0,
    "counts": {
      "concept": 2,
      "definition": 2
    },
    "entities": [],
    "children": []
  }
}
```

`counts` are cumulative for the node and all descendants.

`entities` contains entities whose identifier exactly matches the node path.
For example, `algebra.ring.commutative-unital` may contain the concept entity
and also have descendants such as `definition`, `example`, and `proposition`.

The namespace tree is an editorial browsing structure. It must not be treated as
the full mathematical relation graph.

## Entity Page

Each file in `generated/web/entities/` contains:

```json
{
  "entity": {},
  "outgoing": [],
  "incoming": [],
  "relations": {},
  "history": [],
  "citation_backlinks": []
}
```

`entity` is the normalized source entity, including renderable `display_math` for mathematical entities.

Source entities may include optional publication identifiers:

```json
{
  "doi": "10.1000/example.doi",
  "isbn": "978-0-471-43334-7"
}
```

Source-backed entities include structured `source_refs`:

```json
[
  {
    "source": "source.dummit-foote-abstract-algebra-third-edition",
    "locator": "Chapter 7",
    "note": "Standard abstract algebra treatment of rings."
  }
]
```

`source` is also emitted as a `source_ref` graph edge. `locator` and `note` remain on the entity payload for citation rendering.

`outgoing` and `incoming` are raw graph edges.

`relations` groups common page sections for rendering, such as:

```json
{
  "defined_by": [],
  "definitions": [],
  "examples": [],
  "counterexamples": [],
  "proofs": [],
  "motivating_questions": [],
  "history": [],
  "developed_from": [],
  "developed_into": [],
  "sources": []
}
```

The raw edges remain the authoritative relation data. Relation groups are presentation helpers.

`history` contains first-class historical development entries for pages that are
the subject of historical notes:

```json
{
  "entity": {
    "id": "algebra.ring.commutative-unital.history.integer-arithmetic-abstraction",
    "kind": "historical_note",
    "title": "Rings as an abstraction of integer-like arithmetic",
    "date_label": "Modern abstract algebra formulation",
    "description": "The modern concept...",
    "source_refs": []
  },
  "developed_from": [],
  "developed_into": []
}
```

The historical note entity remains the normative content. `developed_from` and
`developed_into` are summary expansions for presentation.

`citation_backlinks` is populated on source pages. It records every entity that
cites the source, preserving the locator and note from the citing entity:

```json
{
  "source": "source.dummit-foote-abstract-algebra-third-edition",
  "locator": "Chapter 7",
  "note": "Standard abstract algebra treatment of rings.",
  "entity": {
    "id": "algebra.ring.commutative-unital.definition",
    "kind": "definition",
    "title": "Definition of commutative unital ring"
  }
}
```

Source pages may group these backlinks by `locator` for browser presentation.
